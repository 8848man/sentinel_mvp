## Overview

All Gemini calls are made server-side only. The API key is never exposed to clients.  
Two distinct AI operations:

| Operation | Endpoint trigger | Model call |
|-----------|-----------------|------------|
| Metadata Extraction | `POST /incidents/analyze-metadata` | `extract_metadata` |
| Full Incident Analysis | `POST /incidents` (after creation) | `analyze_incident` |

---

## Operation 1: Metadata Extraction

**Input:** `log_text: str`  
**Output:** `suggested_id`, `suggested_title`, `suggested_severity`, `detected_components[]`

### Prompt Template

```python
METADATA_PROMPT = """
You are an incident management assistant.
Analyze the following error log or description and extract structured metadata.

Log input:
---
{log_text}
---

Respond ONLY with valid JSON matching this schema:
{{
  "suggested_title": "short incident title (max 60 chars)",
  "suggested_severity": "critical" | "major" | "minor",
  "detected_components": ["list", "of", "tech", "components"],
  "description": "one-sentence summary of the issue"
}}

Severity rules:
- critical: service down, data loss risk, P0 impact
- major: degraded performance, significant user impact
- minor: low impact, cosmetic, background job issues

Component detection: identify technologies mentioned or implied
(e.g., PostgreSQL, Redis, AWS EKS, Spring Boot, Nginx, Kafka).
"""
```

### Suggested ID Generation

Suggested ID is generated by the backend (not AI):
```python
def generate_suggested_id(year: int, next_seq: int) -> str:
    return f"INC-{year}-{str(next_seq).zfill(3)}"
```

## Operation 2: Full Incident Analysis

**Input:** `log_text`, `title`, `severity`, `components[]`, `similar_incidents[]` (from DB lookup)  
**Output:** `root_cause`, `confidence`, `fix_flows[]`, `similar_incident_codes[]`

### Prompt Template

```python
ANALYSIS_PROMPT = """
You are an expert SRE AI assistant analyzing a production incident.

Incident: {title}
Severity: {severity}
Affected components: {components}

Error logs:
---
{log_text}
---

Known similar past incidents (for context):
{similar_context}

Respond ONLY with valid JSON matching this schema:
{{
  "root_cause": "detailed root cause explanation (2-4 sentences)",
  "confidence": 0.87,
  "fix_flows": [
    {{
      "title": "fix flow name (short, action-oriented)",
      "confidence": 0.96,
      "checklist_items": [
        "Step description 1",
        "Step description 2",
        "Step description 3"
      ]
    }}
  ],
  "similar_incident_codes": ["INC-2026-017", "INC-2026-029"],
  "impact_summary": "one-sentence impact description"
}}

Rules:
- Provide 3-5 fix flows, ordered by confidence descending
- Each fix flow must have 2-5 checklist steps
- confidence is float 0.0–1.0
- similar_incident_codes: return max 3, from the provided context only
- root_cause must be specific and technical
"""
```

## Gemini Service

**File:** `backend/app/services/gemini_service.py`

```python
import google.generativeai as genai
import json

genai.configure(api_key=settings.GEMINI_API_KEY)
model = genai.GenerativeModel("gemini-2.0-flash")

async def extract_metadata(log_text: str) -> dict:
    prompt = METADATA_PROMPT.format(log_text=log_text)
    response = await model.generate_content_async(prompt)
    return parse_json_response(response.text)

async def analyze_incident(log_text, title, severity, components, similar_context) -> dict:
    prompt = ANALYSIS_PROMPT.format(...)
    response = await model.generate_content_async(prompt)
    return parse_json_response(response.text)

def parse_json_response(text: str) -> dict:
    # Strip markdown code fences if present
    clean = text.strip().removeprefix("```json").removeprefix("```").removesuffix("```")
    return json.loads(clean)
```

## Error Handling

| Scenario | Handling |
|----------|---------|
| Gemini API timeout (>15s) | Raise `503` with `"AI service temporarily unavailable"` |
| Invalid JSON in response | Retry once with stricter prompt; if fails, raise `500` |
| Gemini quota exceeded | Raise `503`; log to GCP Cloud Logging |
| Empty response | Raise `500`; log warning |
